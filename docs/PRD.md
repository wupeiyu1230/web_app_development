# 產品需求文件 (PRD) - 食譜收藏夾

## 1. 專案概述
- **背景與動機**：現代人常在網路上看到甚至自己發想出喜歡的食譜，但往往缺乏一個專門統整的平台來有效收藏與管理。同時，有時候打開冰箱不知道能煮什麼，急需一個能根據現有庫存食材來推薦食譜的數位工具。
- **目標用戶**：
  - **一般人**（使用者）：需要收藏、搜尋與發掘新食譜的家庭主婦/夫、料理愛好者、或外宿族群。
  - **管理員**：負責維護平台內容品質、分類與統整食譜資料的人員。
- **核心價值主張**：提供一個操作簡單、直覺的食譜收藏平台，並支援強大的「食材反向搜尋法」，幫助用戶解決「今晚吃什麼」及「如何清理冰箱剩食」的痛點。

## 2. 功能需求
以下為本系統的核心與主要功能（包含使用者故事）：

1. **儲存與建立食譜**
   - 作為 **一般人**，我希望 **能將喜歡的食譜儲存（收藏）或自行建立新的食譜紀錄**，以便 **未來想料理時可以快速找到做法與食材比例**。
2. **瀏覽、搜尋食譜**
   - 作為 **一般人**，我希望 **能透過關鍵字搜尋（例如：紅燒肉）或分類瀏覽食譜**，以便 **快速鎖定我想做的特定料理**。
3. **從食材組合搜尋食譜**
   - 作為 **一般人**，我希望 **能輸入目前擁有的多種食材（如：番茄、雞蛋、豆腐）來搜尋對應的食譜**，以便 **有效消耗剩餘庫存食材**。
4. **帳號註冊與會員系統**
   - 作為 **一般人**，我希望 **能建立專屬的個人帳號並登入系統**，以便 **安全地保存並跨裝置查看我的食譜收藏紀錄**。
5. **後台內容管理功能**
   - 作為 **管理員**，我希望 **能夠查看、編輯、分類或刪除不適當的食譜內容與使用者**，以便 **維持網站資料的正確性與良好品質**。

## 3. 非功能需求
- **技術限制**：
  - 後端將採用 Python Flask 框架。
  - 前端與介面會使用 Jinja2 模板引擎進行渲染。
  - 資料庫採用 SQLite 進行輕量化資料存儲。
- **效能與安全考量**：
  - 使用者密碼必須加密儲存（如使用 bcrypt 雜湊）。
  - 對各種輸入表單需有基本的防護機制，防範 SQL Injection 與 XSS 攻擊。
  - 多食材組合搜尋（Query）需有適當的索引策略，確保搜尋速度。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須具備的雛型功能)
- 使用者註冊與登入/登出功能。
- 食譜庫的基本 CRUD（新增、檢視內容）。
- 基礎關鍵字搜尋食譜。
- 個人收藏夾功能（儲存現有食譜）。
- 依食材組合搜尋食譜功能（系統核心賣點）。

### Should Have (應該具備以完善體驗的功能)
- 簡易的管理員權限與刪除功能。
- 食譜自訂分類標籤（例如：中式、西式、甜點、低卡）。
- 編輯與更新食譜的功能。

### Nice to Have (未來優化方向)
- 允許使用者上傳食譜成品照片。
- 根據食譜材料一鍵匯出「購物清單」功能。
- 食譜評分或留言分享機制。
